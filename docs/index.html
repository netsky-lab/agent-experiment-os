Why it exists

Evals tell you what passed. Experiment OS preserves why it happened.

Failure Taxonomy

Classify planning failures, tool-call syntax drift, context loss, stale API usage, red-green churn, wrong-file edits, and premature completion.

Experiment Memory

Store runs as knowledge: task shape, agent/model/toolchain, observed failures, metric movement, local facts, and confidence.

Decision Policies

Promote repeated evidence into agent-readable policies with provenance, review gates, and dependsOn edges.

System shape

Hypothesis → run → failure → interpretation → intervention → policy.

Hypothesis Test Design Run Observed Failures Metric Movement Interpretation Intervention Policy

For agents

MCP tools present must-load pages, dependsOn graphs, decision rules, issue-derived evidence boundaries, and the next required protocol action before risky edits.

For humans

Dashboard read models expose matrices, clean pass rate, churn, protocol compliance, wiki graph health, policy review queues, and provenance.

Research stance

Not memory for memory’s sake. Evidence becomes useful only after protocol and review.

Issue evidence is not instruction.

GitHub issues become source-backed claims. Agents must verify local package versions and local API surfaces before applying them.

Final pass is not enough.

A run with earlier failed verification is not clean evidence until the failure cause and recovery rationale are recorded.

Policies need provenance.

Accepted policies carry evidence ids, review rationale, confidence, and dependsOn links to failures, interventions, claims, and sources.
